# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for suspected vulnerabilities.

Use GitHub's private vulnerability reporting or security advisory flow when it is
available for this repository. If that private flow is unavailable, open a
minimal public issue asking for a private contact channel without including
technical details, exploit steps, secrets, or proof-of-concept code.

When reporting a vulnerability, include the affected workflow, configuration, or
generated artifact, the impact you observed, and enough reproduction context for
the maintainer to investigate safely.

The maintainer will acknowledge the report, investigate the impact, and
coordinate disclosure or remediation steps as appropriate.
